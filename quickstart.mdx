In this guide, you'll set up Broccoli Queue with Redis and run a complete publish-consume cycle in under 5 minutes. By the end, you'll have published messages to a queue and processed them with concurrent workers.

<Steps>

### Prerequisites

Before starting, ensure you have:

- Rust and Cargo installed (edition 2021 or later)
- A running Redis instance (or start one with `redis-server`)
- Basic familiarity with async Rust and Tokio

### Add Broccoli Queue to Your Project

Add Broccoli Queue to your `Cargo.toml` dependencies:

```toml
[dependencies]
broccoli_queue = "0.4.6"
serde = { version = "1.0", features = ["derive"] }
tokio = { version = "1.0", features = ["full"] }
chrono = { version = "0.4", features = ["serde"] }
```

### Define Your Job Payload

Create a serializable struct for your job data:

```rust
use serde::{Deserialize, Serialize};
use chrono::{DateTime, Utc};

#[derive(Debug, Clone, Serialize, Deserialize)]
struct JobPayload {
    id: String,
    task_name: String,
    parameters: Parameters,
    created_at: DateTime<Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
struct Parameters {
    input_data: String,
    priority: u8,
    timeout_seconds: u32,
}
```

### Create and Configure the Queue

Initialize Broccoli Queue with connection pooling and retry strategies:

```rust
use broccoli_queue::queue::BroccoliQueue;

let queue = BroccoliQueue::builder("redis://localhost:6379")
    .pool_connections(5)
    .failed_message_retry_strategy(Default::default())
    .build()
    .await?;
```

### Publish Messages to the Queue

Send your job payload to a topic:

```rust
let job = JobPayload {
    id: "job-1".to_string(),
    task_name: "process_data".to_string(),
    parameters: Parameters {
        input_data: "data-1".to_string(),
        priority: 1,
        timeout_seconds: 300,
    },
    created_at: chrono::Utc::now(),
};

queue.publish("jobs", None, &job, None).await?;
```

### Consume Messages Concurrently

Process messages with a concurrent worker pool:

```rust
use broccoli_queue::brokers::broker::BrokerMessage;

queue.process_messages(
    "jobs", 
    Some(4),  // 4 concurrent workers
    None,     // default consume options
    |msg: BrokerMessage<JobPayload>| async move {
        println!("Processing job: {:?}", msg);
        // Your business logic here
        Ok(())
    }
).await?;
```

</Steps>

## Complete Publisher Example

```rust
use broccoli_queue::{
    error::BroccoliError,
    queue::BroccoliQueue,
};
use serde::{Deserialize, Serialize};
use chrono::{DateTime, Utc};

#[derive(Debug, Clone, Serialize, Deserialize)]
struct JobPayload {
    id: String,
    task_name: String,
    parameters: Parameters,
    created_at: DateTime<Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
struct Parameters {
    input_data: String,
    priority: u8,
    timeout_seconds: u32,
}

#[tokio::main]
async fn main() -> Result<(), BroccoliError> {
    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .failed_message_retry_strategy(Default::default())
        .pool_connections(5)
        .build()
        .await?;

    let job = JobPayload {
        id: "job-1".to_string(),
        task_name: "process_data".to_string(),
        parameters: Parameters {
            input_data: "data-1".to_string(),
            priority: 1,
            timeout_seconds: 300,
        },
        created_at: chrono::Utc::now(),
    };

    queue.publish("jobs", None, &job, None).await?;
    
    println!("Published job: {}", job.id);
    Ok(())
}
```

## Complete Consumer Example

```rust
use broccoli_queue::{error::BroccoliError, queue::BroccoliQueue, brokers::broker::BrokerMessage};
use serde::{Deserialize, Serialize};
use chrono::{DateTime, Utc};

#[derive(Debug, Clone, Serialize, Deserialize)]
struct JobPayload {
    id: String,
    task_name: String,
    parameters: Parameters,
    created_at: DateTime<Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
struct Parameters {
    input_data: String,
    priority: u8,
    timeout_seconds: u32,
}

#[tokio::main]
async fn main() -> Result<(), BroccoliError> {
    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .pool_connections(5)
        .failed_message_retry_strategy(Default::default())
        .build()
        .await?;

    queue
        .process_messages("jobs", Some(4), None, |msg: BrokerMessage<JobPayload>| async move {
            println!("Processing job: {} ({})", msg.payload.id, msg.payload.task_name);
            // Simulate work
            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
            Ok(())
        })
        .await?;

    Ok(())
}
```

<Note>
Broccoli Queue uses feature flags to enable different broker backends. The Redis backend is enabled by default. To use RabbitMQ or SurrealDB, see the installation guide for feature flag configuration.
</Note>
