Broccoli Queue is a production-ready message queue library for Rust applications — the type-safe, async alternative to Celery. Publish jobs, consume messages, and process workloads concurrently with configurable retry strategies, message scheduling, and fairness queuing across Redis, RabbitMQ, and SurrealDB backends.

## Why Broccoli Queue?

Broccoli Queue brings the ergonomics of task queues like Celery to the Rust ecosystem — with Rust's performance and compile-time type safety. A single, consistent API works across multiple broker backends, so you can start with Redis and switch to RabbitMQ without rewriting your application logic.

## Getting Started

<Steps>
  <Step title="Add the dependency">
    Add `broccoli_queue` to your `Cargo.toml`. Redis support is included by default.
    ```toml Cargo.toml
    [dependencies]
    broccoli_queue = "0.4.6"
    tokio = { version = "1", features = ["full"] }
    serde = { version = "1", features = ["derive"] }
    ```
  </Step>
  <Step title="Build a queue">
    Connect to your broker using the builder API.
    ```rust
    use broccoli_queue::queue::BroccoliQueue;

    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .pool_connections(5)
        .failed_message_retry_strategy(Default::default())
        .build()
        .await?;
    ```
  </Step>
  <Step title="Publish a message">
    Publish any serializable struct to a named topic.
    ```rust
    queue.publish("jobs", None, &my_job, None).await?;
    ```
  </Step>
  <Step title="Process messages">
    Start a concurrent worker loop with a typed handler function.
    ```rust
    queue.process_messages("jobs", Some(4), None, |msg| async move {
        println!("Processing: {:?}", msg.payload);
        Ok(())
    }).await?;
    ```
  </Step>
</Steps>

<Note>
  Broccoli Queue requires Rust edition 2021 and Tokio for async runtime support. See the [Installation guide](/installation) for full setup instructions and feature flag configuration.
</Note>
